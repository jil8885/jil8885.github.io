  1. 정보시스템의 개념과 데이터베이스 ■ 정보 시스템의 정의  데이터를 입력 받아 처리하여 정보를 산출하는 시스템  기업환경에서 업무처리나 경영 의사결정에 필요한 데이터를 수집, 저장, 가공, 배 분하는 시스템  정보시스템은 업무처리나 의사결정 외에도 사원간의 의사소통이나 협력증진, 혹은 통제의 목적으로도 사용  기업환경에서 경쟁력 있는 정보시스템을 구축하기 위해서는 컴퓨터와 통신장비 없이는 불가능하므로, 컴퓨터를 기반으로 하는 인간 기계시스템  기업환경에서 정보시스템은 기업시스템의 하위시스템  정보시스템은 기업경영의 보조적인 역할만을 하는 하위시스템이 아니라, 프로세스 개선을 통해서 고객서비스나 외부파트너와의 협력의 정도를 제고하는 등의 기업 혁신을 가능하게 하는 중요한 하위시스템  기업시스템의 하위시스템으로서 정보시스템은 기업시스템의 목표 달성을 효과적 효율적으로 달성하도록 지원하고 구현하는 목적
  2. 정보시스템의 개념과 데이터베이스 ■ 컴퓨터의 성능 향상과 정보시스템의 탄생 1946년 최초의 컴퓨터인 애니악(ENIAC, Electronic Numerical Integrator And Computer) 개발 1974년 이후 개인 사용자를 위한 PC(Personal Computer)가 보편화 1990년 이후부터는 18~24개월을 주기로 컴퓨터의 성능이 두 배 이상 향상
  3. 정보시스템의 개념과 데이터베이스 ■ 정보시스템을 위한 대용량 저장 장치의 개발 • 플로피디스크와 하드디스크 : 1980년대 이전까지 많이 사용한 저장 장치로 용량이 작음 • CD-ROM : 멀티미디어 데이터가 정보의 주류를 형성하게 되면서 대용량 멀티미디어 데이터 저장의 필요성 증가에 의해 개발 • DVD : CD보다 7배 정도 많은 용량의 데이터를 저장, 최대 17GB까지 저장 가능 • 블루레이 디스크 : DVD보다 5배 많은 용량의 데이터를 저장, 고화질 비디오 데이터의 저장이 가능 • 미래에는 플래시 기억장치와 광 저장 장치들의 발전이 예상됨
  4. 정보시스템의 개념과 데이터베이스 ■ 정보시스템의 형태와 구성  시스템  운영체제, 장치 드라이버, 컴파일러, 코드 라이브러리  애플리케이션  비지니스 업무 등 한 회사 또는 기관의 내부에서 사용하는 시스템  한대, 또는 몇 대의 제한된 하드웨어에서 동시 혹은 순차 동작  인프라  서버 컴퓨터 하드웨어,클라이언트 컴퓨터 하드웨어, 스토리지 장비, 네트워크 장비  보안장비,냉각장치,온도계,공기정화장치 등등  패키지 소프트웨어  패키지화하여 상업적으로 공급 혹은 판매하는 소프트웨어
  5. 정보시스템의 개념과 데이터베이스 ■ 정보시스템의 기능적 종류  ERP 시스템  기업의 전반에 걸친 운용과 관리정보를 지원하는 시스템  시스템의 주요목적은 회사의 여러 기능을 통합하여 효율적인 의사결정을 통하여 비용을 줄이고 이윤을 극대화 시켜줌  기업의 중요한 정보를 보관하고 이용하는데 있어서 전사적인 프레임워크를 구축하여 보안성과 신뢰성 을 높여줌 트랜젝션 처리 시스템  매일 발생하는 비지니스 거래에 의해서 생성되는 정보를 처리하는 시스템  트랜젝션 시스템은 대규모 정보를 처리하는 특징을 가지며 미션중심 시스템  의사결정 지원 시스템  모든 사용자에게 직무관련 정보를 제공하는 시스템  거래에 일어난 데이터를 가지고 비지니스 프로세스를 관리하고 조정할 수 있게 해줌  의사결정 시스템을 이용하여 더 합리적인 의사결정을 하도록 도와줌
  6. 정보시스템의 개념과 데이터베이스 ■ 정보시스템의 처리형태별 종류  일괄처리 시스템  일정시간이나 일정양의 데이터를 모아 한꺼번에 처리하는 시스템  시스템 중심의 자료 처리 방법  테이프와 같은 순차 접근 방법을 사용하는 업무에 적합  단위 시간당 처리하는 작업수가 많으므로 시스템 성능이 높음 ※ 급여 계산, 회계 마감업무, 세무 처리  온라인처리 시스템  데이터 발생, 데이터 처리 요구 시 즉시 처리하는 시스템  사용자 중심의 자료 처리 방법  실시간 처리를 하고, 통신 제어기 필요 ※ 빌링(Billing System), 전력시스템, 보안시스템  분산처리 시스템  분산되어 있는 여러 컴퓨터를 논리적으로 연결하여 하나의 시스템처럼 데이터를 처리하는 것  분산 처리기, 분산 데이터베이스 필요  시스템 구축이 어려운 반면 효용성과 융통성이 높고 시스템 확장이 용이 ※ 가상화페, 인공지능 등
  7. 정보시스템의 개념과 데이터베이스 ■ 정보시스템의 정보와 자료  정보와 자료  정보는 수많은 자료들 중에 필요한 것만을 골라내어 얻는것  정보는 경우에 따라서 전문가들의 손에 의해 자료들을 가공하고 처리해야만 “정보” 로서의 가치를 얻을 수 있는것 예를 들어 우리나라에서 한해 동안 소비되는 담배의 양이 얼마나 되는지를 알기 위해서는 각 시∙도에서 소비되는 담배의 양에 관한 자료를 수집하여 집계를 해야 한다. 이렇게 집계된 결과는 바로 우리가 얻고자 하는 ‘정보’가 되고, 각 시∙도의 담배 소비량은 정보를 얻기 위해 입력한 ‘자료’가 된다.
  8. 정보시스템의 개념과 데이터베이스 ■ 정보시스템과 정보와 자료  자 료  현실 세계로부터 단순한 관찰이나 측정을 통해 수집된 사실이나 값  객관적 실제의 반영이며, 그것을 전달할 수 있도록 기호화한 것 ※ 고객의 주소, 성별, 이름, 나이, 휴대폰 기종, 휴대폰 활용 횟수 등  정 보  자료를 특정한 목적과 문제해결에 도움이 되도록 가공한 것  어떤 목적을 위해 자료를 처리해서 얻을 수 있는 결과  어떤 결정에서 사용자가 목적으로 하는 값  어떤 판단에서 의사결정을 위한 값
  9. 정보시스템의 개념과 데이터베이스 ■ 정보시스템과 데이터베이스와의 관계 정보시스템에서 ACID(원자성, 일관성, 고립성, 지속성)는 데이터베이스 트랜잭션이 안전하게 수행된다는 것을 보장하 기 위한 성질을 가리키는 약어  일관성(Consistency)  트랜잭션이 실행을 성공적으로 완료하면 언제나 일관성 있는 데이터베이스 상태로 유지하는 것을 의미  전통적인 데이터베이스 시스템의 일관성은 휘발성 저장장치와 비 휘발성 저장장치 사이의 데이터 유지에 필요  시스템 장애에 따른 휘발성 저장장치에 저장된 데이터와 비 휘발성 저장장치에 저장된 데이터와의 일관성 유지 는 안전한 데이터 처리에 중요한 이슈  원자성(Atomicity)  트랜잭션과 관련된 작업들이 부분적으로 실행되다가 중단되지 않는 것을 보장하는 능력 예를 들어, 자금 이체는 성공할 수도 실패할 수도 있지만 보내는 쪽에서 돈을 빼 오는 작업만 성공하고 받는 쪽에 돈을 넣는 작업을 실패해서는 안 된다. 원자성은 이와 같이 중간 단계까지 실행되고 실패하는 일이 없도록 하는 것  고립성(Isolation)  트랜잭션을 수행시 다른 트랜잭션의 연산 작업이 끼어들지 못하도록 보장하는 것을 의미한다.  트랜잭션 밖에 있는 어떤 연산도 중간 단계의 데이터를 볼 수 없음을 의미  지속성(Durability)  성공적으로 수행된 트랜잭션은 영원히 반영되어야 함을 의미  시스템 문제, DB 일관성 체크 등을 하더라도 유지되어야 함을 의미
  10. 정보시스템의 개념과 데이터베이스 ■ 정보시스템의 데이터 ■ 정보시스템의 데이터 흐름 (DATA FLOW DIAGRAM)
  11. 정보시스템의 데이터흐름  DFD(Data Flow Diagram)  데이터가 소프트웨어 내의 각 프로세스를 따라 흐르면서 변환 되는 모습을 나타낸 그림으로 소프트웨어 및 정보시스템의 분 석과 설계에서 매우 유용하게 사용되는 다이어그램  데이터 흐름도 또는 자료 흐름도라고 칭함  DFD는 시스템의 모형화 도구로서 가장 보편적으로 사용  데이터에 비해 기능이 매우 복잡하고 중요할 경우에 매우 유용 하게 사용될 수 있음. ■ 정보시스템의 데이터 흐름 (DATA FLOW DIAGRAM)
  12. 정보시스템의 데이터흐름  DFD(Data Flow Diagram) DFD의 구성요소로는 프로세스, 데이터흐름, 데이터저장소, 외부엔 티티 등이 있으며 표기법에 따라 표현하는 그림의 모양이 달라짐 (1)프로세스  프로세스는 입력되는 데이터를 원하는 데이터로 변환하여 출력시키기 위한 과정으로 도형 적 표기형태로는 원(Bubble)과 원안의 이름으로 표현  원안에 기록하는 이름은 아래에 그림과 같이 프로세스가 수행하는 일 또는 프로세스를 수 행하는 행위자를 기술  프로세스는 자체적으로는 데이터를 생성할 수 없고 항상 입력되는 데이터가 있어함  프로세스는 항상 새로운 가치를 부가해야 함 ■ 정보시스템의 데이터 흐름 (DATA FLOW DIAGRAM)
  13. 정보시스템의 데이터흐름 (2)데이터흐름(Data Flow)  데이터흐름(Data Flow)은 DFD의 구성요소들간의 인터페이스를 나타냄  대부분의 데이터흐름은 프로세스들 사이를 연결하지만, 데이터 저장소(Data Store)로부터의 데이터흐름을 나타냄  데이터흐름은 명칭이 부여거나 부여되지 않은 화살표로 표시한다. 단, 후속작업 들의 참조를 위해 되도록 명칭이 부여되는 것이 바람직함  서로 다른 데이터 흐름에는 동일한 이름을 부여하지 않음 ■ 정보시스템의 데이터 흐름 (DATA FLOW DIAGRAM)
  14. 정보시스템의 데이터흐름 (3)데이터저장소(Data Store)  데이터 스토어는 추후의 Access를 위해 데이터를 저장하는 수동적 객체를 말하며, 외부엔터 티와 달리 자신의 액션을 취하지 않으며 데이터 저장이나 Access 요구에 반응하는 역할만 함  데이터저장소(Data Store)는 저장되어 있는 정보 집합  데이터저장소는 테이프, 디스크, 카드 데이터, 스토리지의 Index File 등등일 수도 있음  데이터저장소는 단순한 데이터의 저장을 나타내는 것이지 데이터의 변동을 표시하는 것은 아 님  데이터흐름을 표시함으로써 비로소 데이터의 입출력을 나타냄  데이터 흐름도에서 데이터저장소를 나타내는 표기법은 단순하게 두 개의 직선 즉, 평행선으 로 나타내고, 평행선 안에 데이터저장소의 명칭을 부여함 ■ 정보시스템의 데이터 흐름 (DATA FLOW DIAGRAM)
  15. 정보시스템의 데이터흐름 (4)외부엔터티(External Entity)  외부엔터티는 데이터를 생성, 소비 함으로서 데이터 흐름도(data flow graph)를 주도하는 활성 객체이다. 외부엔터티는 데이터 흐름도 의 입력과 출력에 붙는다. 즉 보통 데이터 흐 름도의 경계에 놓이게 되며, 소스(source)나 싱크(sink)로서 데이터의 흐름을 중단시킴  외부엔터티는 프로세스 처리과정의 데이터발 생의 시작 및 종료를 나타냄  어떤 기업의 내적인(Inside) 외부엔티티는 관 리, 부서, 기능, 시스템등을 포함하며, 기업 외 적인(Outside) 외부엔터티는 고객, 거래처, 공 공기관, 외부시스템 등을 포함  외부엔터티는 데이터 흐름도상에서 프로세스 (Process)와의 상호관련성을 표시하며, 일반적 으로 DFD 범위 밖에 사각형 형태로 표시
